# Caption Blocks Prompt

この文書は、縦型ショート動画の台本を `caption-blocks.json` に変換するための出力契約を定義している。

## Input

入力には時間枠が含まれることがある。時間枠は出力に含まれない。

```txt
【0:00〜0:05】
水面のボートや人を「無料のベビーシッター」代わりにして預けているんだとか。
```

## Output

出力は JSON のみで構成される。説明文、Markdown、コードフェンスは含まれない。

```json
{
  "format": "caption_blocks/v1",
  "blocks": [
    {
      "file_name": "muryo_babysitter",
      "caption": "無料のベビーシッター",
      "narration": "「無料のベビーシッター」"
    }
  ]
}
```

## JSON Contract

- `format` は `caption_blocks/v1` である。
- `blocks` は読み上げ順の昇順に並ぶ。
- `file_name` は番号と拡張子を含まない lower snake の識別名である。
- MP3 ファイル名はプログラムが配列順と `file_name` から生成する。
- `caption` は字幕表示用の必須文字列である。
- `narration` は TTS 用の任意文字列である。

## Caption

- 原文から切り出された表示文字列である。
- 15 文字以内である。
- 画面に出た瞬間に意味が取れる単位である。
- 要約、言い換え、補足を含まない。
- `。`, `、`, `「」`, `()` を含まない。
- `！`, `？` などの感嘆符・疑問符は含めてよい。

## Narration

- 読み上げ文字列が表示文字列と異なる場合のみ存在する。
- 表示用に落とした句読点、括弧、引用符を戻した読み上げ文字列である。
- `。`, `、`, `「」`をそのまま含むことができる。`()` を含まない。

## Segmentation

- 名詞句、述語句、引用句、係り受けは途中で分割されない。
