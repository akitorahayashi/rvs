# rvs

`rvs` is a Bun and TypeScript CLI for preparing and rendering Remotion vertical
shorts.

The current MVP turns captions JSON into narration MP3 files and renders one
captioned MP4 from a project manifest.

## Setup

```bash
bun install
```

## Project Layout

```txt
content/
  reaction_vertical_short/
    active/
      <work>/
        <work>.project.ts
        <work>.captions.json
        narration/
          01_intro.mp3
          02_next.mp3
media/
  bgm/
    music.mp3
  reaction_vertical_short/
    source/
      <source>.mp4
    output/
      <video-name>.mp4
```

Render input is a `<work>.project.ts` path. TTS input is a
`<work>.captions.json` path.
Authored lightweight content stays under `content/`. Heavy source videos and
shared BGM stay under `media/`.
The project manifest stores source and BGM file names only. Source files resolve
from `media/reaction_vertical_short/source/`; BGM files resolve from
`media/bgm/`; outputs resolve to
`media/reaction_vertical_short/output/<video-name>.mp4`.
`<work>.captions.json` uses `tts_format: "caption_narration/v1"` and owns
ordered `file_name`, `caption`, and `narration` blocks. `tts` generates numbered
MP3 files from block order and `file_name`. `narration/` is regenerated by
`tts` and ignored by git.

`prompts/captions.md` contains the prompt for converting rough timed scripts
into `<work>.captions.json`.

## Usage

```bash
bun run serve
bun run rvs tts content/reaction_vertical_short/active/whale/whale.captions.json
bun run rvs render content/reaction_vertical_short/active/whale/whale.project.ts
```

`tts` requires a running VOICEVOX engine at `http://127.0.0.1:50021` by default
and `ffmpeg` on `PATH`. `RVS_VOICEVOX_ENGINE_URL` overrides the engine URL.
`bun run serve` starts the local VOICEVOX engine container. `bun run s` is the
short alias.

Successful renders produce final artifacts under
`media/reaction_vertical_short/output/`. Existing files are not overwritten.
